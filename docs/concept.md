# 🥋 ONE × Sui

# **カジュアル・コンボベッティングアプリ（仮）**

## 🎯 コンセプト

ONE Championship の“瞬間に賭ける興奮”を、Sui の高速性を使って再現する新しいベッティング体験。
従来のレバレッジ／現物ベットでは生まれない **「瞬間の読み合いとコンボ感」** を、Web3 上で実現する。

---

## 🥊 ルール概要（USP）

### **0.6秒コンボ・ベッティング**

* ユーザーがベットした後、**0.6秒以内に次のベット**を行うと
  → **ベット額が自動で2倍になる「コンボ」システム**
* 常に「次の一打」に賭ける格闘技の緊張感を再現
* マーケットに「爆発的な変化（スパイク）」を生み、盛り上がりと熱狂を作る

### **前回ベット（オンチェーン）を参照**

* 各ユーザーの直前ベットのタイムスタンプを参照し
* ベット間隔が 0.6 秒以内なら自動的にコンボ倍率適用
* 他ユーザーのコンボも市場全体にインパクトを与える

---

## ⚡ なぜ「Suiでしかできない」のか？

### **1. 高速ファイナリティ**

* サブ秒レベルのオンチェーン応答が求められる
* **Clock（オンチェーンタイム）**を用いたタイム差判定
  → Ethereum/EVM では不可能なレベルの高速反応

### **2. オブジェクト指向構造**

* 各ユーザーの `Player` オブジェクトに

  * `last_bet_ms`
  * `combo_state`
  * `balance`
    などを保持しやすい

### **3. “瞬間”に依存するゲームロジック**

* ブロック時間ではなく、**ms精度の Clock timestamp**を直接参照できる点が Sui の独自優位性

---

## 👥 ユースケース

* ONE Championship の試合を観ながら
  → 「次の 0.6 秒以内の流れ」に賭ける
* コンボが市場に連鎖し、
  → 試合の盛り上がりとマーケットの盛り上がりが同期する
* ライブ配信 × 即時ベッティングのハイブリッド体験

---

## 💰 ベッティングフロー（簡易）

1. **最初にデポジット（カジノ持ち込み額）を一括で設定**
2. Web クライアント側でベット操作
3. 直前ベットと今回ベットの差分をオンチェーンで評価
4. 0.6 秒以内なら倍率 2 倍でベット成立

※ 内部処理の署名回数は UI で隠蔽（セッション運用など）
※ 技術説明は最小限に留めています

---

## 🎮 ユーザー体験（UX）

* **ワンクリックで即ベット**
  （バックエンド側で署名 UX を可能な限り排除）
* 「コンボ成功！」のアニメーション
* 試合の流れに合わせた“テンポのいい”ゲーム感覚

## 🌐 技術要素（ざっくり）

* `Player` オブジェクトでユーザー状態管理
* デポジット方式で approve の手間を最小化
* Web フロントから高速なベット操作を可能にする仕組み
